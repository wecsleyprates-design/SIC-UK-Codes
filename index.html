01. Problem Statement

Purpose

To explicitly define the misalignment between Worth's US-centric classification standard and the local infrastructure requirements of the UK market.

Worth classifies businesses using the North American Industry Classification System (NAICS) — a US-centric 6-digit code. For UK businesses, the authoritative system is the UK Standard Industrial Classification (UK SIC), a 5-digit code maintained by Companies House.

"The core problem: UK SIC data flows into Worth's pipeline from two vendors (OpenCorporates and Trulioo) but is silently discarded before it reaches any stored or surfaced fact. No uk_sic_code field exists anywhere in the system — not as a resolved fact, not as a database column, not in scoring."

This document explains why, and exactly how to fix it by introducing a jurisdiction-aware fact selector.

05. Layer 2: Fact Engine & "Resolvency"

Purpose

To define the technical logic of arbitration—how the system handles conflicts when multiple vendors disagree on a business's industry.

Fact Resolution Logic

To resolve a fact is the act of picking a single value from multiple candidates. The engine uses a combination of Weight (static trust) and Confidence (dynamic match quality).

// Resolvency logic for jurisdiction-aware facts
class FactEngine {
  resolveUK_SIC(candidates: Fact[]) {
    return candidates
      .filter(c => c.value.length === 5) // Filter to enforce UK SIC 2007 standard
      .sort((a, b) => b.weight - a.weight)[0]; // Highest weight (OC @ 0.9) wins
  }
}
